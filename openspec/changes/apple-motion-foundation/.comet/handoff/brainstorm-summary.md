# Brainstorm Summary

- Change: apple-motion-foundation
- Date: 2026-07-18

## Confirmed Technical Approach

Phased motion program; this change is Phase 1 (foundation + Sheet pilot). User pre-approved ("都同意") the three foundation decisions and delegated autonomous execution through the full comet flow.

- **Tiered substrate**: CSS `linear()` spring easing = zero-dependency default for enter/exit + hover on all components; **Motion** (framer-motion successor) = opt-in `registryDependency` only on gesture-driven components (Sheet first).
- **Spring tokens dual-form**: one `(damping, response)` per preset → emit `--acr-spring-<name>` as CSS `linear()` (~24 stops from the damped-spring ODE) + `--acr-spring-<name>-damping/-response` raw numbers for Motion. Generated by `scripts/gen-springs.mjs` so CSS and JS never drift. Presets: default 1.0/0.4, drawer 0.8/0.3, bounce 0.8/0.4.
- **Skill knowledge split by generality**: general philosophy → `references/apple-motion.md` (attributed, adapted from emilkowalski/skills); acrylic-specific conventions → authored fresh in `rules/motion.md`. materials.md keeps ownership of materials/typography values, citing the principle.
- **Typography**: add `--text-<size>-tracking` + `--text-<size>-leading` companions; large sizes negative tracking/tight leading, body near-zero/comfortable.
- **Accessibility at token layer**: three `@media` blocks (reduced-motion, reduced-transparency, prefers-contrast).
- **Sheet pilot**: Radix Dialog kept for a11y; Motion layered for spring enter/exit (from live value) + 1:1 drag + velocity-projected dismiss/spring-back + scrim dimming; reduced-motion guards the JS path.

## Key Trade-offs and Risks

- `linear()` approximation fidelity / browser support → ~24 stops + `@supports not(linear())` cubic-bezier fallback. `linear()` is baseline in current evergreens.
- Motion bundle weight on Sheet consumers → opt-in only, tree-shakeable `motion/react`.
- Radix presence vs Motion drag unmount timing → `forceMount` content + own exit in Motion `AnimatePresence`; Radix owns a11y/state only.
- Two token files to keep in sync: `registry/acrylic/acrylic.css` (shipped source) AND `app/global.css` (docs-site mirror, line 17).
- Two spring definitions drifting → single generator; numbers are source, `linear()` emitted.

## Testing Strategy

- `openspec validate --strict` for spec integrity.
- `npm run registry:build` (`shadcn build`) → confirm `r/sheet.json` gains `motion` dep + theme item carries new tokens.
- `npm run types:check` / build → no new errors.
- Drive Sheet in running app (`verify` skill): spring enter/exit, interruptible grab-mid-flight, 1:1 drag, velocity-projected dismiss vs spring-back, scrim dimming, reduced-motion degradation.
- Manually confirm tokens hold in light/dark/acrylic with no surface regression.
