A compact "now playing" chip sized for a sidebar footer or any tight rail. It's a shrunk
[Audio Player](/docs/components/audio-player) info group — cover + title + artist with a seek
rail along the bottom edge — plus a single play/pause. Fully controlled; wire the callbacks to
your own audio engine. Renders nothing when idle, so it takes no space until something plays.

## Installation

```bash
npx shadcn add https://acrylic-ui.vercel.app/r/mini-audio-player.json
```

## Usage

Keep a global player (one `<audio>` element) and drop this in a sidebar footer so the current
track and transport stay visible everywhere. `onOpen` (cover/title click) is where you navigate
to the full player. Pass `collapsed` when the sidebar is in its icon-rail state — the chip
shrinks to just the cover plus a progress hairline.

<ComponentPreview name="mini-audio-player-demo" />

```tsx
import { MiniAudioPlayer } from "@/components/acrylic/mini-audio-player"

<MiniAudioPlayer
  track={{ title, artist, cover }}
  playing={playing}
  currentTime={time}
  duration={duration}
  collapsed={sidebarCollapsed}
  onToggle={toggle}
  onSeek={seek}
  onOpen={goToPlayer}
/>
```

Pass `track={null}` (or omit it) for the idle state — the component renders nothing.

## Props

| prop | type | default | description |
| --- | --- | --- | --- |
| `track` | `MiniAudioPlayerTrack \| null` | — | `{ title, artist?, cover? }`; `null`/omitted = idle (renders nothing) |
| `playing` | `boolean` | `false` | play/pause icon state |
| `currentTime` | `number` | `0` | elapsed seconds (seek fill) |
| `duration` | `number` | `0` | total seconds |
| `collapsed` | `boolean` | `false` | icon-rail mode: cover + progress hairline only |
| `onToggle` | `() => void` | — | play/pause clicked |
| `onSeek` | `(seconds: number) => void` | — | seek rail clicked |
| `onOpen` | `() => void` | — | cover/title clicked (navigate to the full player) |

Plus all native `<div>` props.
