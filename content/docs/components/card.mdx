A charcoal translucent pane over a blurred backdrop. Flat at rest (no shadow, no
border); `interactive` adds a hover lift + a soft float shadow. It's a surface
primitive ‚Äî compose list rows, tiles, and panels on top of it.

## Installation

```bash
npx shadcn add https://acrylic-ui.vercel.app/r/card.json
```

## Example ‚Äî card in card

A nested composition (the iOS grouped-list pattern): an outer frosted Card opts
into `nestedSurface` and holds a recessed inner card. The inner surface is **not**
a second blurred Card ‚Äî stacking another `backdrop-blur` over the already-translucent
glass double-samples it and leaves a darkened seam. Instead it's a flat wash that
recesses the surface, reading as a secondary surface on the same pane.

## Example ‚Äî gallery / media card

A media tile with two live segmented controls. **Size** swaps the cover's
`aspect-ratio` (16:9 / 3:4 / 1:1) ‚Äî one source stays uniform while different sources
each keep their own shape, un-cropped ‚Äî and **style** toggles `default` (caption
under the media) against `overlay` (caption on a bottom scrim). Title and subtitle
reuse the Card's own `CardTitle` / `CardDescription` ‚Äî the same type as the
Cover-image-card above ‚Äî so the family reads as one; the corner chip is the acrylic
`Badge` tinted for on-media contrast.

## API Reference

The frosted Card follows the shadcn/ui Card anatomy ‚Äî a surface root plus
Header / Title / Description / Action / Content / Footer sub-parts. Only the root
diverges (it adds `interactive` + `nestedSurface`); every sub-part is a
layout/typography wrapper that renders a `<div>` with a `data-slot` styling hook and
forwards all native `<div>` props (`className`, `onClick`, `style`, ‚Ä¶). `className`
is for layout only ‚Äî color always flows through the `--acr-*` tokens.

### Card

The dark-glass surface root ‚Äî flat at rest (no border, no rest shadow), a soft
float lift on hover when `interactive`. Renders `<div data-slot="card">`.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `interactive` | `boolean` | `false` | Hover lift + soft float shadow, for clickable cards. |
| `nestedSurface` | `boolean` | `false` | Opt child Cards into the nested-surface treatment ‚Äî they drop their own blur and tint one step recessed (the card-in-card pattern). Sets `data-nested-surface`. |
| `className` | `string` | ‚Äî | Merged onto the surface (layout only). |
| `...props` | `React.HTMLAttributes<HTMLDivElement>` | ‚Äî | Forwarded to the root `<div>`. |

```tsx
<Card interactive>
  <CardHeader>
    <CardTitle>Title</CardTitle>
    <CardDescription>Description</CardDescription>
    <CardAction>
      <Button icon variant="ghost" aria-label="More" />
    </CardAction>
  </CardHeader>
  <CardContent>‚Ä¶</CardContent>
  <CardFooter>‚Ä¶</CardFooter>
</Card>
```

### CardHeader

Grid header for the title / description stack; when a `CardAction` is present it
auto-adds a trailing column (`has-data-[slot=card-action]:grid-cols-[1fr_auto]`) to
hold it. Renders `<div data-slot="card-header">`.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `className` | `string` | ‚Äî | Merged onto the header grid. |
| `...props` | `React.ComponentProps<"div">` | ‚Äî | Forwarded to the `<div>`. |
