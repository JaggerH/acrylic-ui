## design.md / Design Doc consistency

Implementation matches design decisions D1–D6. One intentional refinement vs the
Design Doc: the panel motion is applied via Radix `asChild` + a child `motion.div`
rather than `motion.create(SheetPrimitive.Content)` — the latter did not hand Motion
the DOM node (transforms/drag were inert). This is a strictly-better realization of
the same architecture (Radix owns a11y, Motion owns motion) and is documented in the
commit and `rules/motion.md`. No delta-spec contradiction.

## Accepted minor findings (non-blocking)

Sheet hardcodes drawer-preset spring numbers (kept in sync by value); linear()
settle-time vs duration token is perceptually negligible; `SheetContent` omits
Motion-colliding DOM drag handlers from its prop surface; focus-trap releases during
the ~300ms exit animation (inherent to animate-before-unmount, iOS-style).
