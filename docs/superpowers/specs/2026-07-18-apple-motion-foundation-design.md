# Apple Motion Foundation — Technical Design

Deep design for the OpenSpec change `apple-motion-foundation`. The canonical requirements live in `openspec/changes/apple-motion-foundation/specs/`; the high-level architecture in that change's `design.md`. This document is the implementation-depth companion: exact files, the spring math, and the Sheet component structure.

## 1. Spring token generation (`scripts/gen-springs.mjs`)

A damped harmonic oscillator normalized to travel 1→0 (or 0→1) is sampled to produce a CSS `linear()` easing. We parameterize the way Apple/Motion do — **response** `r` (seconds, the natural period-ish settle time) and **damping ratio** `ζ`:

- undamped angular frequency `ω = 2π / r`
- for `ζ < 1` (under/critically): position `x(t) = 1 − e^(−ζω t)·(cos(ω_d t) + (ζω/ω_d)·sin(ω_d t))`, with damped freq `ω_d = ω·√(1−ζ²)`
- for `ζ = 1` (critical): `x(t) = 1 − e^(−ω t)·(1 + ω t)`

The script samples `x(t)` at `N≈24` evenly-spaced `t` over the settle window (until `|1−x| < 0.001` and velocity near zero, capped), normalizes the time axis to `0..100%`, and emits `linear(0, x1 12%, x2 20%, …, 1)`. Easing values may exceed 1 for the bounce preset — that overshoot is the point and `linear()` supports it.

Output shape written into both CSS files:

```css
/* Springs — generated by scripts/gen-springs.mjs from (damping, response). Do not hand-edit the linear() values. */
:root {
  /* default: damping 1.0, response 0.4 — critically damped, no overshoot */
  --acr-spring-default: linear(0, 0.006 3%, 0.045 7%, …, 1);
  --acr-spring-default-damping: 1;
  --acr-spring-default-response: 0.4;
  /* drawer: damping 0.8, response 0.3 */
  --acr-spring-drawer: linear(…);
  --acr-spring-drawer-damping: 0.8;
  --acr-spring-drawer-response: 0.3;
  /* bounce: damping 0.8, response 0.4 — deliberate overshoot for momentum */
  --acr-spring-bounce: linear(…);
  --acr-spring-bounce-damping: 0.8;
  --acr-spring-bounce-response: 0.4;
}
@supports not (transition-timing-function: linear(0, 1)) {
  :root {
    --acr-spring-default: cubic-bezier(0.22, 1, 0.36, 1);
    --acr-spring-drawer: cubic-bezier(0.32, 0.72, 0, 1);
    --acr-spring-bounce: cubic-bezier(0.34, 1.56, 0.64, 1);
  }
}
```

Springs are **theme-independent** (physics, not color) → defined once at `:root`, not per `.dark`/`.acrylic`. Motion maps `(damping, response)` → its `{ type: "spring", bounce, duration }`: `bounce = clamp(1 − damping, 0, 1)` (approx; ζ≥1 → bounce 0), `duration = response`.

## 3. Accessibility media blocks (append to both CSS files)

```css
@media (prefers-reduced-motion: reduce) {
  *, ::before, ::after { --acr-spring-default: linear(0,1); --acr-spring-drawer: linear(0,1); --acr-spring-bounce: linear(0,1); }
  /* components additionally honor this in JS via matchMedia */
}
@media (prefers-reduced-transparency: reduce) {
  .acrylic { --acr-surface: rgba(44,44,48,0.96); --acr-panel: rgba(40,40,44,0.98); /* blur dropped by components reading a flag or via a utility */ }
}
@media (prefers-contrast: more) {
  .acrylic { --acr-border: rgba(255,255,255,0.5); --acr-surface: #1c1c1e; }
}
```

Exact opacity/blur handling refined during build against the running app; the requirement is the behavior, not these literals.

## 4. Sheet refactor

Keep every Radix piece for a11y (`SheetPrimitive.*` = `@radix-ui/react-dialog`: focus trap, scroll lock, ESC, ARIA). Change only how the panel + overlay move.

- **Presence**: wrap content in Motion `AnimatePresence`; `SheetPrimitive.Content` gets `forceMount` so Motion — not Radix's `data-state` keyframes — owns mount/unmount timing. Drop the `data-[state=*]:animate-*` / `slide-*` classes.
- **Enter/exit**: a `motion.div` panel animates `x`/`y` (by `side`) between offscreen and 0 with the drawer spring `{ type:"spring", bounce: 0.2, duration: 0.3 }` from the *live* value, so an interrupt starts where it is. Exit mirrors the entry edge (spatial symmetry).
- **Drag**: Motion `drag="x"|"y"` axis-locked to the side, `dragConstraints` clamped to the open position, `dragElastic` for rubber-band past the closed bound; grab offset is native to Motion's drag.
- **Release** (`onDragEnd` with `info.velocity`): project endpoint `current + (v/1000)·d/(1−d)` (d≈0.998); if past the dismiss threshold or `sign(velocity)` says dismiss → `Sheet` `onOpenChange(false)` and animate closed handing off `info.velocity`; else animate back to 0 (spring).
- **Scrim**: overlay opacity driven by a `useTransform` of the panel's drag offset → dims proportionally.
- **Reduced motion**: a `usePrefersReducedMotion()` hook (matchMedia) swaps the spring for `{ duration: 0.15 }` opacity-only and disables drag-dismiss animation (still closes, just no velocity fling).

New imports: `motion, AnimatePresence, useMotionValue, useTransform` from `motion/react`. Component stays API-identical (`Sheet/SheetTrigger/SheetContent side=…/SheetHeader/…`), so consumers and the Sidebar's mobile off-canvas keep working.
